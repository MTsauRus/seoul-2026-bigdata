## 1. 공모전 평가기준 부합도

공모전 요강의 **"서울 데이터 허브 공공데이터 활용"** 요건에 완전히 부합한다. 사용 데이터 2종 모두 서울 열린데이터광장 공식 데이터다(`서울시 병의원 위치 정보` 12,154건, `서울시 약국 인허가 정보` 5,808건). CP949 인코딩·좌표계(EPSG:5174) 등 원본의 실제 스펙을 그대로 처리한 흔적이 뚜렷해, 데이터 핸들링 진정성이 확인된다.

시각화 부문 4대 기준을 하나씩 따져보면:

- **주제 적절성**: 킥오프 브리프가 규정한 "노인 보행속도 기준 재측정"을 문자 그대로 구현했다. 청년 1.20 m/s → 2,160 m, 노인 0.78 m/s → 1,404 m 라는 물리적 환산까지 코드 상단에 명시되어, 방법론 투명성이 확보됐다.
- **시각화 완성도**: 4개 HTML 모두 인터랙티브(Folium + Plotly)이며 한국어 타이포그래피·툴팁·범례가 일관된다. 다만 **지도 HTML 33MB**는 웹 배포·심사 환경에서 로딩 이슈 가능성이 있어 감점 요소.
- **정보 전달력·유용성**: "손실률 %"와 "접근 시설 개수 감소분"이라는 두 층위의 지표를 병행 제공해, 정책 담당자가 바로 인용 가능한 포맷이다. 특히 **"서울 평균 손실률 55.9%"** 는 기사·보도자료 헤드라인감이다.
- **디자인 독창성**: 상세 평가는 아래 3번 항목 참조.

**요강 부합도 점수: ★★★★☆ (4/5)**

## 2. 팀 주제 정합성 — "노인의 시간으로 본 30분 도보일상권"

**가장 정확히 주제를 물리적으로 구현한 케이스.**
킥오프 브리프가 제시한 핵심 표(청년 30분 vs 65~74세·75~84세·85+세 도달 시간)의 논리를 그대로 **거리 반경 → 접근 시설 수 → 손실률**이라는 3단 파이프라인으로 번역했다.

특히 잘한 지점:

1. **청년선과 노인선을 "한 좌표평면 위"에 병치**한 산점도(02). y=x 기준선과 이론선(0.65² = 0.4225 기울기)을 함께 그려, 단순한 숫자 차이가 아니라 **"물리적 속도차가 면적/접근성에 제곱으로 작용한다"** 는 기하학적 진실을 시각적으로 드러낸다. 이건 브리프가 요구한 "분의 격차"를 한 단계 더 깊게 해석한 결과물이다.
2. **3종(전체 / 병의원 / 약국) 분리**. 브리프가 열거한 "1차 진료기관·약국·보건소" 시설 유형을 개별 분석한 뒤 통합했다. 심사위원이 "종로구는 병의원 손실률 63%지만 약국은 60.9%다" 같은 세부 인사이트를 추출할 수 있게 설계된 점이 돋보인다.

살짝 아쉬운 지점:

- 브리프의 "그 격차 안에 몇 명의 노인이 사는가"라는 **인구 가중 관점이 누락**됐다. 동별 노인 인구 붙이기는 후속 통합 단계에서 필수.
- 핵심 질문 "노인이 만성질환 관리를 동네에서 할 수 있는가"에 대한 답이 **"접근 시설 수"로만 간접 환원**됐다. 시설 수가 3개 이상/이하 같은 정책 임계선(threshold) 시각화가 있으면 더 강해질 것.

**주제 정합도 점수: ★★★★★ (5/5) — 5개 도메인 중 방법론 충실도 최상위 후보.**

## 3. 시각화의 참신성·인사이트 전달력

각 산출물별 소견:

### 3-1. `01_medical_loss_map.html` — 코로플레스 지도 ★★★★☆

- **잘한 점**: 3개 코로플레스(전체·병의원·약국) + 3개 히트맵(시설 분포) + 3개 상위 10동 마커 = 총 9개 레이어를 Folium LayerControl로 토글 가능하게 했다. p10~p90 분위수 기반 컬러맵 정규화는 극단치에 휘둘리지 않게 하는 좋은 선택.
- **참신성**: 코로플레스 자체는 흔하나, "**손실률**"을 코로플레스의 변수로 올린 것이 핵심 차별점. "시설 수 지도"가 아니라 "**불평등 지도**"라는 점에서 한 단계 추상화가 들어갔다.
- **한계**: 브리프가 강조한 **"청년·노인 두 개의 등시선 오버레이"** 는 구현되지 않았다(코로플레스로 대체됨). 만약 시간이 허락한다면 **대표 1~2개 동에 대해 실제 청년권·노인권 원(circle) 두 개를 겹쳐 보여주는 inset**을 추가하면 브리프와 정확히 일치한다.

### 3-2. `02_access_scatter.html` — 청년×노인 산점도 ★★★★★

- 이 팀 전체를 통틀어 **가장 지적인 시각화 중 하나**. 참신성과 통계적 엄격성을 동시에 만족.
- 핵심 아이디어: x축=청년 접근 수, y축=노인 접근 수. **y=x 대각선(손실 0%)** 과 **y=(0.65)²·x 이론선(물리적 속도비 제곱)** 두 기준선을 같이 그려, 점이 어느 선에 가까운가로 "이 동은 기대보다 좋은가/나쁜가"를 한눈에 판정 가능하게 했다.
- 이론선 아래에 떨어진 동들이 바로 **"구조적으로 더 불평등한 동"**이다. 정책 타깃팅에 직결되는 인사이트.
- 약점: 서브플롯 간 x·y축 범위가 달라(전체 3000개 / 병의원 2500개 / 약국 400개) 패널 간 직접 시각 비교는 어렵다. `shared_xaxes` 옵션이나 log-scale 고려 여지 있음.

### 3-3. `03_gu_comparison.html` — 25개 구 바차트 ★★★★☆

- 깔끔한 horizontal bar가 3패널로 정렬. **종로·관악·도봉·용산**이 공통 상위, **강동·금천·마포**가 공통 하위로 나오는 패턴이 즉각 보인다.
- 참신성은 중간. "구별 바차트"는 공모전 단골 구도다. 다만 시설 종류별 3패널로 쪼갠 것이 단조로움을 완화했고, **같은 색 계열(적/녹/보)로 패널을 구분**한 선택은 정체성을 강화했다.
- 개선 여지: **구별 순위가 패널마다 재정렬**되어 "종로구가 병의원·약국 중 어느 쪽에서 더 나쁜가"의 시각적 비교가 약하다. 한쪽 기준(예: 전체)으로 정렬을 고정하고 나머지는 그 순서를 따라가게 하면 패턴 비교력이 올라간다.

### 3-4. `04_distribution.html` — 히스토그램 ★★★☆☆

- 분석자의 **기초 탐색용**으로는 적절하나, 심사위원·대중이 "그래서 뭐?"라고 물을 때 답하기 어려운 형식.
- 손실률이 50~60% 구간에 집중되어 있다는 사실 외에 추가 서사를 만들기 어렵다.
- **개선 제안**: 이 슬롯은 통합 단계에서 다른 시각화(예: **slope chart — 청년 평균 시설 수 → 노인 평균 시설 수를 연결한 기울기 그래프**)로 대체하는 것을 고려할만 하다. 브리프가 명시한 시각화 기법이기도 하다.

**시각화 종합: ★★★★☆ (4/5) — 02번 산점도가 매우 강한 앵커다.**

## 4. 분석 기법의 적합성

### 방법론 체크리스트

| 항목           | 선택                                          | 평가                                                                                       |
| -------------- | --------------------------------------------- | ------------------------------------------------------------------------------------------ |
| 거리 계산      | **유클리드 직선거리** (투영좌표 EPSG:5179 위) | 적절하나 한계 명시 필요 — 도보 네트워크 실거리가 아님. 언덕·하천·지하차도 등 장애물 무시됨 |
| 출발점         | **동 중심점(centroid) 1개**                   | 브리프가 권장한 "격자 다중 출발점"에 미달 — 큰 동일수록 중심점이 실거주지 대표성이 떨어짐  |
| 속도 파라미터  | 청년 1.20 / 노인 0.78 m/s                     | 문헌 표준값. 적절                                                                          |
| 시설 필터링    | 병원분류명 4종 + 약국 영업중·서울 한정        | 타당                                                                                       |
| 좌표계 변환    | WGS84 → EPSG:5179, EPSG:5174 → EPSG:5179      | **정확히 처리**. 좌표계를 섞지 않고 통일한 것은 높은 점수                                  |
| 결측/예외 처리 | 경도>120, 위도>35, 좌표·주소 결측 제거        | 적절                                                                                       |

### 강점

1. **좌표계 엄밀성**: 약국 데이터의 EPSG:5174를 정확히 식별하고 변환한 것은 비전문가가 종종 놓치는 부분이다. GIS 역량 확실.
2. **3종 시설 분리 분석**: 동일 로직을 ftype별로 재사용한 코드 구조가 깔끔.
3. **계산 효율**: centroid 1개 × 시설 벡터화 거리 계산 → O(동수×시설수)로 수초 내 완료.

### 약점 (통합 단계에서 보완 필요)

1. **단일 centroid 한계**: 넓은 동(예: 관악구 신림동)은 동쪽 주민과 서쪽 주민의 실제 접근성이 크게 다르다. **50m 또는 100m 격자 다중 출발점**으로 바꾸면 면적 가중 가능. 브리프가 명시한 방법이기도 하다.
2. **유클리드 거리의 과대추정**: 실제 도보 네트워크 거리는 직선거리 대비 1.2~1.4배. 결과적으로 **현재 손실률이 실제보다 낙관적**일 가능성. 최소한 주의문은 README에 넣어야 한다.
3. **동명 누락**: `dong_name = dong_code`로 임시 처리된 상태. 심사용 최종본에서는 행정동명 매핑 필수(본인도 코드 주석에 명시해둠).
4. **인구 가중 부재**: "노인 인원 몇 명이 영향받는가"가 빠져 있어, 통합 대시보드에서 **노인 인구 × 손실률** 지표 추가 권장.

**분석 기법 점수: ★★★★☆ (4/5) — 구현 엄밀성은 상위권, 격자 출발점·네트워크 거리는 후속 과제.**

### 한 줄 총평

> **"브리프의 방법론을 가장 충실히 코드로 번역한 파트. 02번 산점도는 팀 통합 발표의 대표 앵커로 쓸 만큼 강력하다. 다만 단일 centroid·유클리드 거리·인구 가중 부재 세 가지는 통합 단계의 보편 과제로 남는다."**

### 통합 단계에 반영할 액션 아이템

1. **02번 산점도는 팀 메인 덱에 채택 후보.** y=x 기준선 + 0.65² 이론선의 해석을 발표 스크립트로 작성할 가치 있음.
2. **01번 지도에 대표 동 2~3개 inset(청년권·노인권 이중 원)을 추가**해 브리프의 "등시선 오버레이"와 정합시키기.
3. **04번 히스토그램은 slope chart로 교체 검토.** 동일 슬롯에서 인사이트 전달력 향상 가능.
4. 동명 매핑 CSV 붙이기 · 격자 출발점 재계산 · 노인 인구 가중 합성 — 이 3가지는 D3뿐 아니라 **5개 도메인 공통 작업**으로 넘겨야 함.
5. 33MB 지도 HTML은 **coordinates precision 축소 + 심사용 PNG 스냅샷 병행**으로 배포 가능성 확보.
