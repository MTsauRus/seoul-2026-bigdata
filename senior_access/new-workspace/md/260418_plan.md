## Part A. 프로젝트 당위성 시각화 — 청년 vs 노인 보행 도달 영역

### A-0. 왜 이 시각화가 프로젝트의 근간인가

2040 서울도시기본계획의 **"도보 30분 자족 생활권"** 약속. 그 30분은 누구의 시간인가? 이 질문을 데이터와 지도로 **한 장으로** 답하는 게 이 시각화다.

- 이 시각화 없으면 나머지 4명의 도메인 분석이 공중에 뜬다 ("왜 노인 보행속도로 재측정하나?"에 답 못 함)
- 이 시각화만 제대로 되면 나머지 도메인은 자동으로 정당화됨
- 발표 슬라이드에서 **가장 앞**에 나와야 함 (WHY 슬라이드)

#### 노인 보행속도 근거 자료 (발표용 인용)

- **한국 노인 평균 보행속도: 0.66 m/s** (Korean Living Profiles of Older People Survey, 2008-2011, n=8,000; Korean Journal of Rehabilitation Medicine)
- 서구 건강 노인(Bohannon 1997): 75세+ 평균 약 0.78 m/s
- 85세+ 약 0.58 m/s (Montero-Odasso et al. 2005)
- **본 프로젝트 채택값**:
  - 청년: **1.2 m/s** (4.32 km/h, Bohannon 20-40세 평균)
  - 노인: **0.66 m/s** (한국 65세+ 지역사회 거주자 평균)
  - 보조: 0.78 m/s (건강 노인), 0.58 m/s (고령 노인) — 감도 분석용

#### Step 5. 등시선(Isochrone) 계산

```python
import networkx as nx
from shapely.geometry import Point
from shapely.ops import unary_union
import numpy as np

def compute_isochrone(G, start_lon, start_lat, time_col, cutoff_sec):
    """
    time_col: 'time_young' or 'time_senior'
    cutoff_sec: 30분 = 1800초
    """
    start_node = ox.distance.nearest_nodes(G, start_lon, start_lat)
    travel = nx.single_source_dijkstra_path_length(
        G, start_node, cutoff=cutoff_sec, weight=time_col)

    # 도달 가능 노드 좌표로 alpha shape 생성
    points = [Point(G.nodes[n]['x'], G.nodes[n]['y']) for n in travel.keys()]
    hull = unary_union(points).convex_hull.buffer(0.0005)  # 초기엔 convex hull, 이후 alpha shape 전환
    return hull, travel

# 각 샘플 지점 × 2 속도 = 6~10개 등시선
```

> **구현 팁**: `alphashape` 라이브러리 사용 시 convex hull보다 실제 형태 근접. `alpha=300` 권장 (좌표 단위 degree). 초기엔 convex hull로 prototyping, 나중에 alpha shape로 교체.

### A-3. 시각화 활용 방안

#### 시각화 A1. **"한 지점에서의 충격" — 등시선 오버레이 지도** (메인 킥)

**무엇을 보여주나**:

- 지도 위에 단일 출발점에서의 청년 30분권(옅은 파랑)과 노인 30분권(진한 빨강)을 겹쳐 그림
- 차이 면적(도넛 모양)에 음영 처리 → **"잃어버린 영역"** 을 시각적으로 충격적으로 표현
- 출발점 마커 + 노인 등시선 경계선 안쪽의 주요 시설(경로당·복지관·지하철역 등) 점 표시
- 토글 버튼: 지점 1~5 선택 가능

**왜 유의미한가**:

- 발표 첫 30초 안에 청중 설득 가능
- 숫자·퍼센트 없이도 "노인에게는 다른 도시" 라는 메시지 전달
- 종로 지점은 격차가 작고 강북·관악 지점은 크다 → **같은 서울이 아니다** 가 드러남

**Interactive 요소**:

- 지점 드롭다운
- 속도 슬라이더: 청년(1.2) / 건강노인(0.78) / 일반노인(0.66) / 고령노인(0.58)
- 시간 슬라이더: 15분 / 30분 / 45분

#### 시각화 A2. **"한 끼의 차이" — 면적 손실 비교 바 차트**

**무엇을 보여주나**:

- 5개 지점 × 각 지점의 청년 30분권 면적 vs 노인 30분권 면적
- 수평 막대 (gauge bar처럼 청년 막대 위에 노인 막대 중첩)
- 손실 면적을 `km²` + `㎢당 평균 65+ 인구` 라벨 → **영향 인원**까지 표시

**왜 유의미한가**:

- A1의 시각적 충격을 수치로 뒷받침
- "강북구 미아역 기준, 노인은 청년의 X% 면적에만 도달 가능. 이 격차에 거주하는 65세 이상 추정 인원 N명"

#### 시각화 A3. **"서울 전체 격차 히트맵" — 동별 평균 손실률 코로플레스**

**무엇을 보여주나**:

- 서울 426개 행정동 각각의 중심점에서 등시선 2종을 계산하고, 동별 면적 손실률을 색상으로 표현
- Plotly의 choropleth_mapbox
- hover 시 동명·손실률·65+ 인구 표시

**왜 유의미한가**:

- A1이 샘플 지점이라면 A3은 서울 전체 체계적 측정
- 나머지 팀원들의 도메인 분석이 왜 동별로 필요한지 정당화

**주의**: 426개 등시선 계산은 시간 부담. dev 단계는 25개 자치구 중심점으로 prototyping, 최종은 멀티프로세싱+캐시로 전체 산출. 계산 성공 못 하면 A1+A2만으로도 충분.

## Part B. 기후 도메인 시각화 — 폭염·한파·결빙

### B-0. 도메인 주제

노인은 기후 위기에 가장 취약한 집단이다. 서울 2040 플랜(p.20-23)도 폭염·폭설·한파를 5대 재해로 지정. 본 도메인은 세 시각화로 "노인의 기후 위기 × 보행" 를 입체적으로 보여준다:

1. **B1. 폭염** — 무더위쉼터에 걸어 닿을 수 있는가?
2. **B2. 한파** — 한파쉼터에 걸어 닿을 수 있는가?
3. **B3. 결빙** — 겨울 빙판에서 노인은 어디가 위험한가? (**신규 아이디어**)

B1·B2는 쉼터 좌표 + 노인 보행 등시선으로 직관적 분석. B3가 프로젝트의 **숨겨진 차별화 카드**: 여름 폭염 쉼터 분석은 흔하지만, 겨울 빙판 보행 위험을 지도화한 노인 대상 분석은 드물다.

### B-3. 시각화 활용 방안

#### 시각화 B1. **"폭염 정오, 갈 곳 없는 노인" — 무더위쉼터 도달 듀얼 패널 지도**

**무엇을 보여주나**:

- Left 패널: 서울 전역 무더위쉼터 점 지도 + 노인 15분 보행 등시선 합집합(연두색 음영)
- Right 패널: 도달 불가 영역(회색) + 독거노인 인구 hexbin 오버레이(진한 빨강 = 독거노인 밀집 + 쉼터 미도달 = 고위험)
- Hover: 동명 + 독거노인 수 + 15분 내 쉼터 수

**왜 유의미한가**:

- 서울 내 쉼터가 "많다"는 인상 → 노인 보행 15분 기준으로 보면 실제로는 사각 다수
- 독거노인 밀집 지역 + 쉼터 미도달 = **생명 직결**

**Interactive 요소**:

- 쉼터 클릭 → 해당 쉼터에서 도달 가능한 영역 강조
- 보행속도 슬라이더 (0.58 / 0.66 / 0.78 / 1.20)
- 시간 슬라이더 (5 / 10 / 15 / 20분)

#### 시각화 B2. **"한파 새벽, 숨을 곳 없는 노인" — 한파쉼터 + 독거노인 오버레이**

B1과 구조 동일하되 한파쉼터(OA-21066) 데이터로 구성. 추가 메시지:

- 한파쉼터는 무더위쉼터보다 수가 적음 → 사각이 더 큼
- Hover에 "이 동의 75세+ 독거노인 X명, 15분 내 한파쉼터 Y개"

#### 시각화 B3. **"빙판의 도시" — 결빙 위험 히트맵 (차별화 카드)** ⭐

**무엇을 보여주나**:

- 서울 전역 행정동별 `ice_risk_score` 코로플레스 (회색~진한 파랑 계열)
- 제설함 위치 점 오버레이 (작은 흰 점)
- 상위 20개 위험 동 마커 강조 + 라벨
- Toggle 가능한 레이어:
  - 레이어 1: 제설함 부재 비율
  - 레이어 2: 노인 보행사고 발생률 (자치구 기준)
  - 레이어 3: 종합 결빙 위험 점수

**왜 유의미한가**:

- **공모전에서 본 적 없는 각도**: 여름 폭염만 다루고 겨울 결빙은 다루지 않는 경향. 프로젝트 차별화
- 심사위원 반응 예상: **"결빙? 맞네, 이거 왜 안 했지?"** 라는 무릎 탁 반응
- 정책 연결 직관적: 제설함 미설치 구간 중 노인 밀집 동 → 겨울철 우선 제설 루트 재검토 근거
- 시의성: 겨울 낙상 → 노인 대퇴골 골절 → 의학적으로 1년 내 사망률 20%+ (WHO). 데이터상 공공보건 이슈.

**Interactive 요소**:

- 레이어 토글 (3종)
- 상위 N개 위험 동 하이라이트 슬라이더 (5/10/20)
- Hover 시 동명 + 제설함 수 + 노인 보행사고 건수

#### 시각화 B3-보조. **"겨울 한 달, 노인 보행사고 누적 패턴" (선택 — 시간 여유 시)**

자치구별 월별 노인 보행사고 건수 히트맵 (X축 월, Y축 자치구). 12-2월 정점 시각화. 단순하지만 B3 메인을 뒷받침.
