## 0. 메타

- **주제**: "끊어진 서울, 멀어진 물가 — 선형 장벽과 수변 접근성의 정량화"
- **좌표계**: 거리계산 EPSG:5179, 시각화 EPSG:4326
- **스택**: Python, OSMnx, GeoPandas, NetworkX, Shapely, matplotlib, Folium, plotly

## 🚨 대회 존립조건 (절대 준수)

**"서울 데이터 허브 서비스 활용 내용"이 제출물 필수항목.** 단순 "1건 이상 사용"이 아니라 활용 데이터/기능/화면 구체 기재 필요. **OSM 단독이면 탈락.** 이 프로젝트의 주력 데이터는 서울 데이터 허브 5종, OSM은 보조 기하 소스.

## 1. 대목표

서울의 선형 장벽(지상철도·노후 간선도로)이 만드는 보행 단절과, 수변 접근성 저하를 정량·시각화하여, 2040 서울플랜의 철도 지하화·입체복합화 정책 우선순위 의사결정을 돕는다.

근거: 2040 플랜 Goal 3(기반시설 입체복합화, 지상철도 101.2km 지하화) + 노후 간선도로 지하화 + 수변감성도시 축.

## 2. 소목표 3개

- **A. 장벽 정량화**: 지상철도+장벽도로의 구간별 단절 강도를 **Detour Ratio**(실제 보행거리/직선거리)로 수치화
- **B. 수변 접근성**: 집계구→가장 가까운 수변공원 **실제 도달시간** vs 직선 이상시간 격차로 "벽에 막혀 물가 못 가는 지역" 식별
- **C. 정책 시뮬레이션**: 우회비 상위 구간 입체화 시 Before/After → 우선순위 권고

## 3. 핵심 결정사항 (Decisions)

1. 단절 지표 = Detour Ratio, 접근성 지표 = 도달시간. **억지 통합 금지**
2. 수변은 "단절" 아닌 "접근성"으로 처리
3. 장벽도로 조작적 정의: `highway=motorway/trunk` + 보행 crossing 간격 >500m
4. 지상철도: 공식 GIS 없음 → OSM 기하 + 2040 101.2km 크로스검증
5. 수변 진입점: 수변 폴리곤 경계 ∩ walk 노드 자동추출
6. 인구가중: OA-14979 집계구 1차, OA-14991 행정동 폴백
7. OSM 필터 고정: `tunnel=yes OR layer<0 OR covered=yes` 제거
8. **신분당선 substring 버그**: `name == "신분당선"` 완전일치, 분당선·신분당선 모두 지상후보에서 제외 (100% 지하)
9. 복복선 inflation: `name` 기준 dissolve 후 length 계산
10. **데이터 주종**: 서울 데이터 허브 주력, OSM 보조

## 4. 서울 데이터 허브 필수 5종 (주력)

| ID       | 데이터셋                    | 역할                |
| -------- | --------------------------- | ------------------- |
| OA-21210 | 서울시 육교 공간정보        | 횡단점 (A·B·C)      |
| OA-22353 | 서울시 보도육교 위치현황    | 횡단점 (A)          |
| OA-21213 | 서울시 지하철역 연계 지하도 | 횡단점 (A)          |
| OA-14979 | 서울 생활인구 집계구 단위   | 인구가중 (B·C 핵심) |
| —        | 서울 행정동 경계            | 배경/조인키         |

→ 발표자료 필수 섹션: **"서울 데이터 허브 활용 내용"** (데이터셋 ID, 화면 스크린샷, 전처리, 기여도)
→ 프로젝트 내내 `docs/data_hub_usage.md`에 로그 축적
